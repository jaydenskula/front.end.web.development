Present in this assignment: - Input type "submit" - Input type "number" - Input type "datetime-local" - Input type "color" - 'select' and 'option' This was another simple assignment. I had a bit of trouble figuring out the Javascript syntax so that it takes the user input and manipulates the page directly based on the input, however with some time I figured it out. I did not accomplish as much as I wanted to for this project. I can't help but be a bit disappointed with myself, however I do genuinely like my idea and I want to continue to work on it. I can easily say that a big takeaway from this project is that I have a lack of understanding of scope and scale. Learning a completely new JavaScript library, especially when I do not have extensive experience with JavaScript by itself, was daunting. For this project specifically, I used the animation timeline feature Anime JS offers. The library offers so many different components and options so I hope that, as I use it more often, I will get a chance to use its other features and get more comfortable with it overall.

My creative and technical process for this project was somewhat straightforward. While there was an artistic piece in my project, the JS code was the most important to implement. After the code was fully functional, I created my final image assets for aesthetic and visual satisfaction. The critique day that we had in class was also helpful, as I got cool recommendations on how to enhance the experience and I did my best to fulfill the feedback that was given to me. I had a lot of fun with this project. ENTRY 12 | PROJECT 2 WIP

We have made significant progress on our project so far. During Thanksgiving break, Wilson implemented the Pokemon API. During a class, Ana did some of the styling for our site so each Pokemon displayed as their own "cards" in a grid. In my own time, I used react to add interaction to our site. Each Pokemon has a "more info" button that displays details of the Pokemon that was clicked. The additional information disappears if the button is clicked again. I ran into some formatting issues which were solved in class. I also had an issue that all the info was displayed at once when one of the buttons was clicked. This was solved by putting the react function in a separate jsx file.

Wilson and I spent a good amount of time doing more styling for our site. We wanted the information cards to look like actual Pokemon cards. We were having a little bit of trouble with the images because they all appeared to be different sizes. We realized that the issue was caused by the fact that our css defined maximum height instead of just the height. Some images did not reach the maximum height and therefore displayed in different sizes. Our site adapts to mobile screens. In terms of our goals, we have reached all of them other than the design.
